# Documentation Guidelines — Measure

Ecosystem-wide conventions: see `docs/GUIDELINES.md` at the workspace root.
This file documents conventions specific to the measure component.

---

## Page map

| Page | Purpose |
|------|---------|
| `README.md` | Package positioning and quick start. Also the docs landing page. |
| `guides/usage.md` | Three-step workflow: prepare catalog → write YAML config → run analysis. |
| `guides/configuration.md` | Full YAML schema reference: DATA, MEASUREMENT, OUTPUT blocks + per-model parameters. |
| `guides/api_reference.rst` | Auto-generated from source. Do not hand-edit. |

| `methodology/demo_*.ipynb` | Cross-cutting methodology notebooks (A/A testing, parameter sensitivity). |

## Tutorials

### Structure

Every method demo notebook follows this sequence:

1. **Title and overview** — what the tutorial demonstrates and why it matters.
2. **Setup** — imports and any prerequisite configuration.
3. **Worked example** — step-by-step walkthrough with code and output.
4. **Inspection** — examine the results in detail.

### Execution

All method and methodology notebooks are executable (`nbsphinx_execute = "always"`).
No external API keys required — all demos use synthetic data generated by the
Online Retail Simulator.

### Naming

Method demo notebooks: `demo_{method_name}.ipynb` (e.g. `demo_experiment.ipynb`).
Configs for notebooks: `methods/configs/{method_name}.yaml`.
